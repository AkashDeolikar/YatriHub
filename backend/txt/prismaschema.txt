NOT:

hotel_backend

So when you run:

Docker says:

service "hotel_backend" is not running

Because that service name does not exist.

✅ Correct Command

Use the SERVICE name (app), not container name.

Run this:

docker compose exec app npx prisma migrate dev --name add_food_order

OR if you want reset:

docker compose exec app npx prisma migrate reset
🧠 Why This Happened

In docker-compose.yml you likely have:

services:
  app:
    container_name: hotel_backend

So:

hotel_backend = container name

app = service name

Docker Compose uses service name for exec, not container name.
